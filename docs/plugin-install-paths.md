# Plugin Installation Paths: Claude Code vs Claude Cowork

## Overview

Forge Plugin Manager installs plugins into two targets with **different directory structures and path formats**.

## Claude Code (CLI)

**Base:** `~/.claude/plugins/` (all platforms, home directory)

```
~/.claude/
├── settings.json                        # enabledPlugins: { "forge-core@reumbra": true }
└── plugins/
    ├── known_marketplaces.json          # marketplace registry
    ├── installed_plugins.json           # installed plugin versions
    ├── cache/
    │   └── reumbra/
    │       └── {plugin}/{version}/      # cached plugin files (active copy)
    └── marketplaces/                    # (for github-sourced; directory-source uses external path)
```

### known_marketplaces.json entry

```json
{
  "reumbra": {
    "source": { "source": "directory", "path": "/absolute/path/to/marketplace" },
    "installLocation": "/absolute/path/to/marketplace",
    "lastUpdated": "2026-03-08T22:30:00.000Z"
  }
}
```

- **Paths are absolute** (platform-native)
- `source.path` points to a directory containing `.claude-plugin/marketplace.json`
- Claude Code reads marketplace.json, finds plugin sources, copies to `cache/`

### installed_plugins.json entry

```json
{
  "forge-core@reumbra": [{
    "scope": "user",
    "installPath": "/home/user/.claude/plugins/cache/reumbra/forge-core/6.2.0",
    "version": "6.2.0",
    "installedAt": "2026-03-08T22:30:00.000Z",
    "lastUpdated": "2026-03-08T22:30:00.000Z"
  }]
}
```

### settings.json — enabledPlugins

```json
{
  "enabledPlugins": {
    "forge-core@reumbra": true,
    "forge-qa@reumbra": true
  }
}
```

### Marketplace directory (source)

Currently at `~/.forge/marketplace/` (legacy) or OS-standard config dir:
- Windows: `%APPDATA%/forge-devkit/marketplace/`
- macOS: `~/Library/Application Support/forge-devkit/marketplace/`
- Linux: `~/.config/forge-devkit/marketplace/`

```
marketplace/
├── .claude-plugin/
│   └── marketplace.json                 # name, owner, plugins[]
└── plugins/
    ├── forge-core/
    │   ├── .claude-plugin/plugin.json
    │   ├── skills/
    │   ├── commands/
    │   └── hooks/
    └── forge-qa/
        └── ...
```

## Claude Cowork (Desktop App)

**Base:** `%APPDATA%/Claude/local-agent-mode-sessions/{sessionId}/{userId}/cowork_plugins/`

- `{sessionId}` — UUID of the Cowork session (e.g. `4774e195-3b88-4166-8387-101b32fe6ecc`)
- `{userId}` — UUID of the user account (e.g. `6b3860a3-5481-481f-8de0-f6d4c088fdb0`)

```
cowork_plugins/
├── known_marketplaces.json              # marketplace registry (same schema as Code)
├── installed_plugins.json               # installed plugins (same schema as Code)
├── .install-manifests/
│   └── {plugin}@{marketplace}.json      # per-plugin install manifest
├── cache/
│   └── reumbra/
│       └── {plugin}/{version}/          # cached plugin files
└── marketplaces/
    └── reumbra/
        ├── .claude-plugin/
        │   └── marketplace.json
        ├── forge-core/
        │   ├── .claude-plugin/plugin.json
        │   ├── skills/
        │   └── commands/
        └── forge-qa/
            └── ...
```

### Key difference: RELATIVE paths with `mnt/` prefix

All paths in Cowork JSON files use **relative paths** prefixed with `mnt/`:

```json
{
  "reumbra": {
    "source": { "source": "directory", "path": "mnt/.claude/cowork_plugins/marketplaces/reumbra" },
    "installLocation": "mnt/.claude/cowork_plugins/marketplaces/reumbra",
    "lastUpdated": "2026-03-08T22:30:00.000Z"
  }
}
```

```json
{
  "forge-core@reumbra": [{
    "scope": "user",
    "installPath": "mnt/.claude/cowork_plugins/cache/reumbra/forge-core/6.2.0",
    "version": "6.2.0",
    "installedAt": "2026-03-08T22:30:00.000Z",
    "lastUpdated": "2026-03-08T22:30:00.000Z"
  }]
}
```

### No settings.json

Cowork does **not** use a separate `settings.json` with `enabledPlugins`. Plugins are enabled by being present in `installed_plugins.json`.

### .install-manifests/

Per-plugin JSON files (e.g. `forge-core@reumbra.json`). Created by Cowork during install. May need to be generated by the manager for full compatibility.

## Detection Logic

### Claude Code
- Check: `~/.claude/` directory exists
- Path: `home_dir().join(".claude")`

### Claude Cowork
- Check: `%APPDATA%/Claude/local-agent-mode-sessions/` exists and has session subdirs
- Path: scan for `{sessionId}/{userId}/cowork_plugins/` pattern
- Session/user IDs: enumerate directories (there is typically one session with one user)

### Windows specifics
- `%APPDATA%` = `C:\Users\{user}\AppData\Roaming`
- Cowork runs natively on Windows, uses Windows paths internally
- But JSON files use `mnt/` prefix (internal to Cowork's sandbox/runtime)

## Summary Table

| Aspect | Claude Code | Claude Cowork |
|--------|-------------|---------------|
| Base dir | `~/.claude/plugins/` | `%APPDATA%/Claude/.../cowork_plugins/` |
| Path format | Absolute | Relative (`mnt/` prefix) |
| Marketplace location | External dir (registered) | Inside `cowork_plugins/marketplaces/` |
| Plugin cache | `cache/{marketplace}/{plugin}/{ver}/` | `cache/{marketplace}/{plugin}/{ver}/` |
| Enable mechanism | `settings.json` → `enabledPlugins` | `installed_plugins.json` (presence = enabled) |
| Install manifests | None | `.install-manifests/{plugin}@{mkt}.json` |
| Config files to update | `known_marketplaces.json`, `installed_plugins.json`, `settings.json` | `known_marketplaces.json`, `installed_plugins.json` |
